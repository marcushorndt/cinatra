# Security Policy

Cinatra is an early open source release and is **not production ready**. Even so, we take security seriously and welcome responsible disclosure.

## Reporting a vulnerability

Please report security vulnerabilities **privately** by email to **security@cinatra.ai**.

Do **not** open a public GitHub issue, pull request, or discussion for a security vulnerability — that exposes the problem before a fix is available.

In your report, please include where you can:

- a description of the issue and its impact,
- the steps or a minimal proof of concept to reproduce it,
- the affected components, routes, or configuration, and
- any suggested remediation.

## What to expect

- We aim to acknowledge new reports within a few business days.
- We will work with you to validate the issue and agree on a remediation timeline.
- This is a best-effort process: there is no paid bug bounty and no formal response-time guarantee yet.
- Please give us a reasonable opportunity to ship a fix before public disclosure.

## Scope

The current development line of this repository is in scope. Because Cinatra is pre-release software, security hardening is ongoing — see the not-production-ready notice in the [README](README.md).

For the engineering view of the platform's authorization and trust model, see [security](https://docs.cinatra.ai/references/platform/security/).
