## Attribution-record recovery

The original `34a830a` bump commit and a follow-up correction were pushed
directly to `main`. The post-merge `truthful-attribution-gate` requires a
verifiable verification arm (a human `Reviewed-by`, or a `Gate-suite` +
`Accountable` gate arm that the gate can verify in PR context); a no-PR direct
push can carry neither verifiably. This record lands through a pull request so
the gate arm is verified in PR context, restoring a green `main`. The published
`v0.1.5` tag and image are immutable and unchanged: `34a830a^{tree}` is
identical to this PR's merge tree, so artifact provenance is preserved.
